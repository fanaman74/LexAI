# LexAI Phase 4 (API Endpoints + Rich UI) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Implement the full document/case/search API surface (instruct.md §15–17) and the six frontend pages (§18): dashboard, documents list, document detail, search, cases list, case detail.

**Architecture:** Next.js 16 App Router. All API routes are `app/api/**/route.ts` using `requireUser()` + the cookie-scoped Supabase client (RLS enforces ownership). Keyword search uses a new `keyword_search_documents()` SQL function (ts_rank + ts_headline snippets); hybrid search merges keyword + semantic scores in TypeScript. Pages are server components with small client components for interactivity (filters, search form, case assignment).

**Tech Stack:** Next.js 16, TypeScript, Tailwind CSS, @supabase/ssr, Postgres tsvector + pgvector.

**Conventions for all tasks (read these files before coding):**
- Auth: `lib/auth/session.ts` → `requireUser()` (redirects) for pages; for API routes use `getUser()` and return 401 JSON if null (don't redirect from APIs).
- Supabase: `lib/supabase/server.ts` → `await createClient()` (cookie-scoped, RLS applies).
- Storage: `lib/storage/storage.ts` → `signedUrl(supabase, path, expiresInSeconds)`.
- Audit: `lib/audit/audit.ts` → `logAudit(supabase, userId, action, opts)`.
- Existing patterns: `app/api/documents/upload/route.ts`, `app/api/search/semantic/route.ts`, `app/documents/page.tsx`.
- Verify each task with `npx tsc --noEmit` (fast) and finish the phase with `npm run build`.

**Out of scope (later phases):** AI endpoints (`/api/ai/*` — Phase 5; render disabled "Ask AI" buttons), case bundle export (Phase 6), tag management UI (defer; tags tables exist but no UI yet).

## Task 1: Migration — `keyword_search_documents()` + reset-for-reprocess helper

**Files:**

- [ ] **Step 1: Write migration**

```sql
-- Phase 4: ranked keyword search with snippets + reprocess reset helper

create or replace function public.keyword_search_documents(
    search_query text,
    filter_user_id uuid,
    filter_case_id uuid default null,
    filter_source_type text default null,
    match_count int default 20
)
returns table (
    document_id uuid,
    original_filename text,
    display_title text,
    source_type text,
    document_date date,
    ai_short_summary text,
    rank real,
    snippet text
)
language sql stable
security definer
set search_path = public
as $$
    select
        d.id,
        d.original_filename,
        d.display_title,
        d.source_type,
        d.document_date,
        d.ai_short_summary,
        ts_rank(d.search_vector, websearch_to_tsquery('english', search_query)) as rank,
        ts_headline('english', coalesce(d.extracted_text, ''),
                    websearch_to_tsquery('english', search_query),
                    'MaxWords=40, MinWords=20, MaxFragments=2') as snippet
    from documents d
    left join case_documents cd on cd.document_id = d.id
    where d.user_id = filter_user_id
      and d.search_vector @@ websearch_to_tsquery('english', search_query)
      and (filter_case_id is null or cd.case_id = filter_case_id)
      and (filter_source_type is null or d.source_type = filter_source_type)
    group by d.id
    order by rank desc
    limit match_count;
$$;

-- Reset a document so the dispatcher re-processes it (used by POST /reprocess)
create or replace function public.reset_document_for_reprocess(doc_id uuid, requesting_user uuid)
returns void
language plpgsql
security definer
set search_path = public
as $$
begin
  update documents
  set processing_status = 'queued',
      chunking_status = null,
      processing_error = null,
      updated_at = now()
  where id = doc_id and user_id = requesting_user;
end;
$$;
```

## Task 2: Documents API routes

**Files:**
- Create: `app/api/documents/route.ts` (GET list)
- Create: `app/api/documents/[id]/route.ts` (GET detail, DELETE)
- Create: `app/api/documents/[id]/chunks/route.ts` (GET)
- Create: `app/api/documents/[id]/reconstruct/route.ts` (GET)
- Create: `app/api/documents/[id]/reprocess/route.ts` (POST)
- Create: `app/api/documents/[id]/signed-url/route.ts` (GET)

All routes: `const user = await getUser(); if (!user) return NextResponse.json({ error: "Unauthorized" }, { status: 401 });` then cookie-scoped client (RLS scopes rows to the user automatically — still add `.eq("user_id", user.id)` belt-and-suspenders on writes). Next.js 16 dynamic params: `{ params }: { params: Promise<{ id: string }> }` then `const { id } = await params;`.

Contracts:

**GET /api/documents** — query params: `source_type`, `processing_status`, `case_id`, `date_from`, `date_to`, `q` (ilike on original_filename), `limit` (default 50), `offset`. If `case_id` given, first fetch `case_documents.document_id` list for that case, then `.in("id", ids)`. Select: `id, original_filename, display_title, source_type, processing_status, chunking_status, document_date, created_at, file_size_bytes, parent_document_id, ai_short_summary`. Order `created_at desc`. Return `{ documents, count }` (use `{ count: "exact" }`).

**GET /api/documents/[id]** — select `*` single; also fetch children: `id, original_filename, source_type, processing_status` where `parent_document_id = id`; and parent (`id, original_filename`) if `parent_document_id` set. 404 if not found. Return `{ document, parent, children }`.

**DELETE /api/documents/[id]** — fetch doc first (404 if missing). Delete storage objects: collect `storage_path`, `markdown_storage_path` (if set), then `supabase.storage.from(LEGAL_BUCKET).remove(paths)`. Then `.delete().eq("id", id)` (chunks cascade). `logAudit(supabase, user.id, "delete", { documentId: id, metadata: { filename } })`. Return `{ deleted: true }`.

**GET /api/documents/[id]/chunks** — verify doc ownership (fetch id only, 404 if missing), then chunks ordered by `chunk_index asc`, select `chunk_id, chunk_index, content, token_count, char_count, section_title, metadata`. Return `{ document_id, chunk_count, chunks }`.

**GET /api/documents/[id]/reconstruct** — instruct.md §16: fetch document (`id, original_filename, source_type, display_title`), 404 if missing; fetch chunks ordered by `chunk_index asc` selecting `chunk_id, chunk_index, content, content_markdown`; markdown = join of `content_markdown ?? content` with `"\n\n"`. If no chunks, fall back to the document's `markdown_text` column. `logAudit(..., "reconstruct", ...)`. Return `{ document, chunks: [{chunk_id, chunk_index}], markdown }`.

**POST /api/documents/[id]/reprocess** — call `supabase.rpc("reset_document_for_reprocess", { doc_id: id, requesting_user: user.id })`. `logAudit(..., "reprocess", ...)`. Return `{ status: "queued" }`.

**GET /api/documents/[id]/signed-url** — query param `kind` = `"original"` (default) | `"markdown"`. Fetch doc, pick `storage_path` or `markdown_storage_path` (400 if markdown requested but null). `signedUrl(supabase, path, 300)`. `logAudit(..., "download", ...)`. Return `{ url, expires_in: 300 }`.

## Task 3: Keyword + hybrid search API routes

**Files:**
- Create: `app/api/search/keyword/route.ts`
- Create: `app/api/search/hybrid/route.ts`
- Reference: `app/api/search/semantic/route.ts` (existing — reuse its response shaping)

**POST /api/search/keyword** — body `{ query, case_id?, source_type?, limit? = 20 }`. Validate query ≥ 2 chars. Call `supabase.rpc("keyword_search_documents", { search_query, filter_user_id: user.id, filter_case_id, filter_source_type, match_count: limit })`. Map rows to the §17 shape: `{ results: [{ document_id, original_filename, display_title, source_type, document_date, ai_short_summary, best_score: rank, matched_chunks: [], snippet }] }`. `logAudit(..., "search", { metadata: { mode: "keyword", query } })`.

**POST /api/search/hybrid** — body `{ query, case_id?, limit? = 20 }`. Run keyword RPC and the semantic flow (embed via `EMBED_SERVER_URL/embed` then `match_document_chunks` RPC — copy the pattern from the semantic route) **in parallel** with `Promise.all`. If the embed server is down, degrade gracefully: return keyword-only results with `"mode": "keyword-only"` in the response instead of failing. Normalize: keyword rank → divide by max rank in the set; semantic similarity is already 0–1. `combined_score = 0.5 * norm_keyword + 0.5 * best_semantic` (missing side contributes 0). Merge by `document_id`, enrich missing doc metadata from a single `.in("id", ids)` query, sort by combined_score desc, slice to limit. Return `{ query, mode, results: [{ document_id, original_filename, display_title, source_type, document_date, ai_short_summary, best_score: combined_score, keyword_score, semantic_score, matched_chunks, snippet }] }`. `logAudit(..., "search", { metadata: { mode: "hybrid", query } })`.

## Task 4: Cases API routes

**Files:**
- Create: `app/api/cases/route.ts` (GET list, POST create)
- Create: `app/api/cases/[id]/route.ts` (GET, PATCH, DELETE)
- Create: `app/api/cases/[id]/documents/route.ts` (GET, POST)
- Create: `app/api/cases/[id]/documents/[documentId]/route.ts` (DELETE)

Contracts:

**GET /api/cases** — list user's cases ordered `created_at desc`, plus per-case document count (fetch `case_documents` rows for the user's cases and count in TS, or use `select("*, case_documents(count)")`). Return `{ cases }`.

**POST /api/cases** — body `{ name, description? }`. Validate name non-empty. Insert with `user_id: user.id`, status `'active'`. `logAudit(..., "case_create", { caseId })`. Return `{ case }` 201.

**GET /api/cases/[id]** — single case, 404 if missing. Return `{ case }`.

**PATCH /api/cases/[id]** — body any of `{ name, description, status }` (`status` ∈ active|archived). Update, return `{ case }`.

**DELETE /api/cases/[id]** — delete the case row (case_documents cascade; documents untouched). `logAudit(..., "case_delete", { caseId: id })`. Return `{ deleted: true }`.

**GET /api/cases/[id]/documents** — join: fetch case_documents for case, then documents `.in("id", ids)` selecting list-page fields. Return `{ documents }`.

**POST /api/cases/[id]/documents** — body `{ document_id }`. Verify both case and document exist (404). Upsert into `case_documents` (`case_id, document_id, user_id`) ignoring duplicates (`onConflict: "case_id,document_id", ignoreDuplicates: true`). `logAudit(..., "case_assign", { caseId, documentId })`. Return `{ assigned: true }`.

**DELETE /api/cases/[id]/documents/[documentId]** — delete the join row. `logAudit(..., "case_unassign", ...)`. Return `{ removed: true }`.

Note: if the `AuditAction` type in `lib/types/index.ts` lacks these action strings, extend it (`"case_create" | "case_delete" | "case_assign" | "case_unassign" | "reprocess" | "reconstruct" | "search" | "download" | "delete"` — keep existing members).
